1. Glossar

BegriffDefinition
AthletEin registrierter Endanwender des Systems. Steht bei "Feels Like Organic" unter Vertrag und erfasst seine Leistungsdaten.
Betreuer / TrainerEin Benutzer mit erweiterten Rechten (z.B. ein Mitarbeiter von "Feels Like Organic" wie Simon Zweigler), der Athleten verwalten und Berichte einsehen kann.
TrainingseinheitEin einzelner, vom Athleten erfasster Datensatz zu einer sportlichen Aktivität (Laufen, Rad fahren oder Schwimmen). Besteht mindestens aus Datum, Sportart, Dauer und Distanz.
WettbewerbEin spezieller Datensatz, der eine offizielle Wettkampfteilnahme des Athleten dokumentiert (z.B. Marathon, Triathlon).
ProduktEin von "Feels Like Organic" hergestelltes und vertriebenes Produkt (z.B. Sportgel), das im System hinterlegt ist.
Das SystemBezeichnet die in diesem Projekt zu entwickelnde Webanwendung zur Erfassung und Analyse von Leistungsdaten der Athleten.
RankingEine dynamisch generierte, sortierte Liste von Athleten, basierend auf einer bestimmten Leistungsmetrik in einem bestimmten Zeitraum.

    NummerUC-07
    PaketSchritt 4: Angrenzende Systeme
    AutorIsabella Schwarz
    Version1.0
    KurzbeschreibungDer Athlet kann über die Login Seite sein Passwort zurücksetzen durch Angabe der Email Adresse. Über ein zugandten Link kann ein neues Passwort angelegt werden
    beteiligte Aktoren (Stakeholder)Athlet
    FachverantwortlicherAtussa Mehrawari (Product Owner)
    ReferenzenUC-01 Kontoverwaltung, UC-02: Athleten-Verwaltung/Einladung
    Vorbedingungen-Benutzer besitzt registriertes Konto mit gültiger Email Adresse. -Der Mail Server ist Verfügbar. -Benutzer wählt “Passwort Vergessen“ Option aus
    Nachbedingungen-Passwort wurde erfolgreich geändert und in der Datenbank aktualisiert. -Benutzer kann sich mit dem neuen Passwort einloggen. -Das temporäre Zurücksetzungslink wird nach Nutzung ungültig.
    Typischer Ablauf1. Benutzer klickt auf “Passwort Vergessen” auf der Login Seite. 2. Benutzer gibt seine registrierte Email an 3. System prüft ob die Email existiert. 4. System sendet einen einmaligen Zurücksetz-Link per E-Mail. 5. Benutzer öffnet den Link und gibt ein neues Passwort ein. 6. System speichert das neue Passwort und bestätigt den erfolgreichen Vorgang.
    Alternative Abläufe2a. Benutzer gibt eine nicht registrierte E-Mail-Adresse ein. 4a. Passwortzurücksendungs E-Mail kann aufgrund eines technischen Fehlers nicht gesendet werden. 5a. Der Link ist abgelaufen oder bereits genutzt
    KritikalitätHoch, Funktion ist sicherheitsrelevant und notwendig für den Zugriff auf das Benutzerkonto.
